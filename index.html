✦ The Core Framework

The CRATE Framework

Every powerful prompt follows this structure. We invented CRATE to make it easy to remember and apply consistently across any task.

C
Context
Set the scene. Who are you? What situation are you in? What background does the AI need to understand before it can help?
R
Role
Tell the AI who to be. A senior developer, a marketing copywriter, a Socratic teacher — the role dramatically shapes tone, depth, and vocabulary.
A
Action
The core task. Be precise: not "write something about X" but "write a 200-word summary of X for a non-technical audience, focusing on business impact."
T
Target
Define the audience and the intended use. Who will read this? Where will it appear? What should they think or do after consuming it?
E
Examples
Show don't just tell. Include samples of the format, tone, or style you want. Even one example shifts AI output quality significantly.

Tokenization

AI models don't read words — they read tokens. A token is roughly 3–4 characters of English text. "Unbelievable" is 3 tokens. "cat" is 1. This has real implications for how you write prompts.

Token limits define how much an AI can read in one go. GPT-4o handles 128K tokens (~96,000 words). Claude handles up to 200K. Understanding tokens helps you manage long documents, estimate API costs, and avoid truncation errors.

# Token counts for common items: "Hello world" → 2 tokens "I love prompting" → 4 tokens "supercalifragilistic" → 7 tokens 1 page of text ≈ 400–500 tokens 1 novel ≈ 100,000 tokens
💡 Tip: Use tiktoken (OpenAI's library) or Anthropic's token counter to estimate token usage before sending expensive API calls.

Temperature & Randomness

Temperature is a number (usually 0 to 2) that controls how "creative" or "random" the AI is when picking the next token. At temperature 0, the model always picks the single most-probable next token. At temperature 2, it samples wildly from the distribution.

This is one of the most misunderstood settings. Most people leave it at default, but for professional use, you should always choose deliberately.

# Recommended temperature settings: temperature = 0 → Deterministic facts, extraction temperature = 0.3 → Technical writing, code temperature = 0.7 → Good all-purpose default temperature = 1.0 → Creative writing, brainstorming temperature = 1.5 → Highly experimental / poetic
⚠️ Warning: High temperature + factual questions = hallucinations. Always lower temperature when accuracy matters.

Chain-of-Thought Prompting

Research by Google Brain (Wei et al., 2022) showed that simply adding "Let's think step by step" to a prompt increased math accuracy dramatically. This works because it forces the model to allocate tokens to intermediate reasoning, not just jump to an answer.

You can use zero-shot CoT (just tell it to think step by step) or few-shot CoT (show it examples of step-by-step reasoning).

# Zero-shot Chain-of-Thought: "What is 17% of 340? Think through this step by step before answering." # Few-shot Chain-of-Thought: "Q: A train travels 60mph for 2.5 hrs. How far does it travel? A: Step 1: distance = speed × time Step 2: 60 × 2.5 = 150 Answer: 150 miles. Q: [your actual question here]"
💡 Pro move: Ask the model to show its work even when you don't need it — then check the reasoning for errors before trusting the final answer.

Context Windows

The context window is the total amount of text an AI can "see" in one conversation — both the input you send and the output it generates. Everything outside this window is completely invisible to the model. It literally cannot remember it.

Modern models have massive context windows, but longer context doesn't mean better reasoning. There's a well-documented "lost-in-the-middle" phenomenon where models perform worse at recalling information buried in the center of very long contexts.

# Context window comparison (2025): GPT-4o → 128K tokens (~96K words) Claude 3.5 → 200K tokens (~150K words) Gemini 1.5 Pro → 1M tokens (~750K words) Llama 3.1 → 128K tokens
⚠️ The "lost in the middle" problem: Put your most important information at the START or END of long prompts — never bury critical instructions in the middle.

System Prompts

A system prompt is a special message sent to the AI before the user conversation begins. It defines the AI's persona, rules, knowledge constraints, response style, and hard limits. It's invisible to end-users in most products — but incredibly powerful.

Think of it as the job description + employee handbook you give the AI before it starts work. Everything in the system prompt has elevated authority compared to user messages in most modern models.

# Example system prompt skeleton: You are [NAME], a [ROLE] specialized in [DOMAIN]. Your audience is [WHO]. Always: - [Positive behavior 1] - [Positive behavior 2] Never: - [Prohibited action 1] - [Prohibited action 2] Respond in [TONE]. Keep answers [LENGTH]. Format as [FORMAT].
💡 System prompts compound. A well-written system prompt reduces the length and complexity needed in every individual user message.

Retrieval-Augmented Generation (RAG)

RAG is the technique of fetching relevant documents from an external knowledge base and injecting them into the prompt context before the AI generates a response. This lets you ground AI outputs in verified, up-to-date information — eliminating hallucinations about facts in your data.

This is how most serious enterprise AI products work. Knowing RAG at even a basic level makes you far more effective as both a prompt engineer and an AI product builder.

# Simplified RAG flow: 1. User asks a question 2. System embeds question → vector 3. Search vector database for similar docs 4. Retrieve top K relevant chunks 5. Inject chunks into prompt as context: "Using only the following info: [RETRIEVED DOCS] Answer: [USER QUESTION]" 6. AI generates grounded answer
💡 Even without a vector database, you can do "manual RAG" — copy-pasting relevant documents into your prompt. For many tasks, this is all you need.
✦ Real Examples

Bad prompts vs. great prompts

See the exact difference in quality with side-by-side comparisons of weak and strong prompting for the same tasks.

✗ Weak Prompt Writing Task
"Write me an email about the meeting."
No context, audience, tone, length, purpose, or goal. You'll get a generic, useless template.
✓ Strong Prompt Writing Task
"Write a 3-paragraph follow-up email to a potential B2B client (mid-size logistics company) after our first sales meeting. Tone: professional but warm. Include: a brief recap of their pain points (manual tracking, errors), how our software solves them, and a clear CTA to schedule a product demo. Avoid jargon. End with a genuine line about enjoying the conversation."
Context, audience, tone, structure, length, CTA, constraint — all specified. Output will be usable on first try.
✗ Weak Prompt Analysis Task
"Summarize this document."
What length? For whom? What aspects matter? You'll get a random, probably too-long, generic summary.
✓ Strong Prompt Analysis Task
"You are a senior business analyst. Summarize the attached report in exactly 5 bullet points for a busy C-suite executive who has 2 minutes to read this. Each bullet should be one clear sentence. Focus only on: key findings, financial implications, and recommended actions. Use plain language — no jargon."
Role, format, audience, constraints, focus — the AI has everything it needs to produce exactly what you want.
✗ Weak Prompt Code Task
"Fix my code."
No language, no error message, no code provided, no explanation of expected behavior. Impossible to help effectively.
✓ Strong Prompt Code Task
"I have a Python 3.11 function that should filter a list of dictionaries by a given key-value pair. It works for strings but fails silently for integers — it just returns an empty list. Here's the code: [code]. Expected input: filter_by(data, 'age', 30). Expected output: all dicts where age == 30. Identify the bug, explain why it happens, then provide the corrected function."
Language version, expected behavior, failing behavior, code, example I/O — and a structured ask (explain + fix).
✦ 15+ Techniques

Every technique you'll master

From day-one basics to advanced professional methods — with a clear level indicator for each.

Beginner 🎯
Role Prompting
Assign the AI a specific expert identity. "Act as a UX researcher" or "You are a Harvard law professor" activates domain-specific vocabulary, reasoning patterns, and depth.
Beginner 📝
Instruction Layering
Build prompts with multiple nested instructions: primary task → constraints → format → tone. Each layer adds precision. The order matters — most important instructions first and last.
Beginner 🔄
Iterative Refinement
Treat prompting as a dialogue, not a one-shot transaction. Follow up with: "Make it shorter," "More formal," "Redo the second paragraph focusing on ROI." Build toward your target.
Intermediate 🧪
Contrastive Prompting
Tell the AI both what to do AND what not to do. "Write a product description. Do NOT use exclamation marks, avoid hype words like 'revolutionary', and don't mention competitors."
Intermediate 🪞
Reflection & Self-Critique
Ask the AI to evaluate its own output: "Review your answer and identify any weaknesses, then rewrite with those fixed." This reliably improves output quality without you doing the reviewing.
Intermediate 🎲
Multiple Persona Output
Ask for the same topic from multiple perspectives: "Give me 3 versions of this headline — one for millennials, one for Gen X executives, one for technical engineers." Instant A/B testing.
Intermediate 🔍
Decomposition Prompting
Break impossible tasks into solvable sub-tasks. "First, list the 5 main stakeholders. Then, for each stakeholder, identify their primary concern. Finally, draft a plan addressing each concern."
Intermediate 📐
Format Anchoring
Show the exact output format you want — including empty brackets, placeholder text, and structural markers. The AI will fill in your template precisely, reliably matching your schema.
Advanced 🌐
Tree of Thought (ToT)
An advanced extension of CoT where the model explores multiple reasoning branches simultaneously, evaluates them, and backtracks if a branch fails. Dramatically better for complex planning problems.
Advanced 🗳️
Self-Consistency Voting
Run the same prompt multiple times with higher temperature, then choose the answer that appears most frequently. Particularly effective for math, logic, and ambiguous analysis tasks.
Advanced 🔗
Prompt Chaining
Pass the output of one AI call as the input to the next. Research → outline → draft → edit → SEO-optimize — each step with a specialized prompt, producing output far beyond what one prompt can achieve.
Advanced 🤖
Meta-Prompting
Ask the AI to write a better prompt for you. "I want to prompt an AI to do [task]. Write me the best possible prompt for this." Then use that prompt. Recursive and surprisingly effective.
Advanced 📊
Constitutional Prompting
Give the AI a set of explicit principles (a "constitution") it must follow when generating and then self-evaluating its output. Based on Anthropic's Constitutional AI research.
Advanced 🧬
Prompt Injection Defense
Understand how malicious input can hijack AI behavior — and how to write system prompts that are robust against injection attacks in production applications.
Advanced ⚡
Structured Output Forcing
Use JSON mode, XML tags, and schema constraints to guarantee parseable, consistent output from every API call — eliminating the need for fragile string parsing.
